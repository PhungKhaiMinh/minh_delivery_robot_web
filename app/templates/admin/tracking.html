Thiếu Admin.opt_map trong file .db

RTAB-Map Viewer hiển thị lưới occupancy 2D lưu sau tối ưu graph (cột Admin.opt_map). File map trên server không có blob này (NULL) — web buộc dựng PNG từ điểm laser (point_accum), nên hình dạng / độ đặc không thể giống y chang viewer.

Để khớp viewer: mở cùng file .db trong RTAB-Map Desktop → Graph View / tối ưu graph → File → Save (hoặc xuất DB) để ghi opt_map vào bảng Admin, rồi dùng file đã lưu cho RTAB_MAP_DB_PATH.

Đã tăng ngân sách điểm laser và làm đặc raster khi thiếu opt_map — chỉ là xấp xỉ. Đã sửa lỗi trộn tọa độ: khi không có opt_map, graph và laser chỉ còn dùng Node.pose gốc (không lẫn opt_poses).

Đồ thị đường đi robot (waypoint)

Cạnh waypoint–waypoint (tím): bật Chế độ nối cạnh → chọn waypoint A → waypoint B.
Cạnh pickup ↔ waypoint (cam nét đứt): pickup được coi như đỉnh trên mạng — chọn pickup rồi waypoint (hoặc ngược lại) để robot chỉ đi vào mạng qua các cạnh bạn nối. Hoạch định đơn test (Orders) chạy Dijkstra trên tổng đồ thị đó, không tự nối ảo pickup nữa. Nhấn lại cùng một điểm để hủy bước chọn. Esc xóa lựa chọn.

Nút Lưu ghi cả cạnh WP–WP và pickup↔waypoint. Tải pickup để vẽ điểm cam và nối portal. Điểm tím (waypoint) và cam (pickup) dùng cùng phép chiếu với graph RTAB trên map để trùng tọa độ m với PNG/opt_map. Chế độ nối cạnh chỉ trên tab RTAB 2D. Tab Đám mây PLY 3D chỉ hiển thị điểm tham chiếu cùng tọa độ x,y.
